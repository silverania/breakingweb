L' utilità principale di un ambiente virtuale Python , è quella di preservare l' ambiente Python che usa il sistema operativo . Usando l' ambiente python di sistema , invece che uno virtuale , per le proprie applicazioni ed esperimenti vari , ci si espone al rischio di corrompere il sistema operativo ! pensiamo a distribuzioni linux come Gentoo , in cui il gestore di pacchetti : il Portage , appunto , scritto interamente in Python , a quali rischi sarebbe esposto , in caso di corruzione dell' ambiente Python di sistema ! soluzione a ciò e non solo : Creare un ambiente Python virtuale ridondante con un proprio eseguibile Python , su cui possiamo fare tutti gli esperimenti che vogliamo. A partire dalla versione 3.5 di python l' uso del modulo venv nella creazione di ambienti virtuali python è raccomandata .
Assicuriamoci di aver installato Python sul sistema : da un terminale eseguite il comando python

mario@androgenic:~/$ python

L' output nel terminale dovrà mostrare la versione di python che per , l' uso del modulo venv dovrà essere maggiore o uguale a 3.5
per versioni inferiori a 3.5 di python va usato pyvenv

Creiamo l' ambiente virtuale Python :

Abbiamo creato l' ambiente virtuale Python , tuttavia da terminale se eseguo il comando python , va in esecuzione ancora la versione python usata dal sistema operativo , quella installata nel percorso /usr/bin/python* . Per raggiungere il nostro scopo , che è quello di non usare la versione python in /usr/bin , per non rischiare di corrompere il sistema , come detto : dobbiamo attivare nel terminale l' ambiente virtuale , in modo che quando lanceremo python dalla riga di comando , verrà eseguita la versione Python virtuale dal percorso : $ /home/username/cartellaDellAmbienteVirtualePython : per fare ciò dobbiamo eseguire il comando : $ /home/username/cartellaDellAmbienteVirtualePython/bin/activate dall' interno della cartella virtuale che abbiamo creato in precedenza , il comando activate modificherà il $PATH istruendolo ad eseguire la versione python dell' ambiente virtuale , al posto di quella che usa il sistema operativo in /usr/bin/python*

A partire da ora ogni volta che avvierete l' eseguibile python dal terminale , userete l' eseguibile dell' ambiente virtuale . Guardate come il comando activate ha modificato due variabili di sistema molto importanti : PATH , PS1 . Nello screenshot sotto , il terminale è diviso in due sezioni , quella di sopra e quella di sotto. Nella sezione di sopra è stato attivato l' ambiente virtuale Python, sotto invece no . Guardate come sono diversi i valori delle variabili : PATH e PS1 nelle rispettive sezioni .
