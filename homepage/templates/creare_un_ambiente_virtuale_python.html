L' utilità principale di un ambiente virtuale Python , è quella di preservare l' ambiente Python che usa il sistema operativo . Usando l' ambiente python di sistema , invece che uno virtuale , per le proprie applicazioni ed esperimenti vari , ci si espone al rischio di corrompere il sistema operativo ! pensiamo a distribuzioni linux come Gentoo , in cui il gestore di pacchetti : il Portage , appunto , scritto interamente in Python , a quali rischi sarebbe esposto , in caso di corruzione dell' ambiente Python di sistema ! soluzione a ciò e non solo : Creare un ambiente Python virtuale ridondante con un proprio eseguibile Python , su cui possiamo fare tutti gli esperimenti che vogliamo. A partire dalla versione 3.5 di python l' uso del modulo venv nella creazione di ambienti virtuali python è raccomandata .
Assicuriamoci di aver installato Python sul sistema : da un terminale eseguite il comando python

mario@androgenic:~/$ python

L' output nel terminale dovrà mostrare una versione python adatta all' uso del modulo venv che dovrà essere maggiore o uguale a 3.5 . Su versioni inferiori non è consigliato usare venv e probabilmente neanche possibile.

per versioni inferiori a 3.5 di python va usato pyvenv

Creiamo l' ambiente virtuale Python

Abbiamo creato l' ambiente virtuale Python , tuttavia da terminale se eseguo il comando python , va in esecuzione ancora la versione python usata dal sistema operativo , quella installata nel percorso /usr/bin/python* . Per raggiungere il nostro scopo , che è quello di non usare la versione python in /usr/bin , per non rischiare di corrompere il sistema , come detto : dobbiamo attivare nel terminale l' ambiente virtuale , in modo che quando lanceremo python dalla riga di comando , verrà eseguita la versione Python virtuale dal percorso : $ /home/username/ambienteVirtualePython : per fare ciò dobbiamo eseguire il comando :
$ ~/AmbienteVirtualePython/bin/activate che attiverà appunto l' ambiente Virtuale. Attenzione perchè l' ambiente virtuale che attiveremo con il comando activate , sarà valido solo nel terminale dov'è stato appunto attivato . Se apro un'altra istanza del terminale o avvio python da un posto diverso dal terminale in cui è stato attivato : verrà avviato l' eseguile che usa il sistema operativo in : /usr/{bin,lib}/python*. Il comando activate modificherà la variabile $PATH contenuta in $USER/.bashrc , istruendolo ad eseguire la versione python dell' ambiente virtuale , al posto di quella che usa il sistema operativo in /usr/bin/python*

Attiviamo L' Ambiente Virutale

nell' immagine giù si vede come il comando activate abbia modificato una variabile di sistema molto importante PATH . Giù Nello screenshot , il terminale è diviso in due sezioni : sopra e sotto. Nella sezione di sopra è stato attivato l' ambiente virtuale Python, sotto invece no . Guardate come è diverso il valore della variabile PATH nelle rispettive sezioni .
